Quantum Circuit Cutting

Overview

  • We used wire cutting to split a 7-qubit circuit into two 4-qubit circuits, allowing broader circuit decomposition at the cost of added overhead.
  • This method uses Move instructions—functionally like swap gates—to transfer quantum state information between the two circuit halves via an ancillary circuit.
  • Each subcircuit is executed separately, once on a simulator and once on IBMQ’s quantum hardware.
  • The resulting sub-experiment data is used to build Quasiprobability Distributions, which are then combined to reconstruct the expectation value of the original 7-qubit circuit.

Tech Stack

  • Python – For writing quantum circuit logic, estimation functions, and handling data
  • Qiskit – IBM's open-source quantum computing SDK used to – Build and cut quantum circuits, Execute on simulators (AerSimulator) and real quantum hardware (IBMQ backend), and Use Estimator for expectation value calculations
  • IBM Quantum (IBMQ) – Cloud-accessible quantum hardware for running real quantum circuits
  • Qiskit Runtime / Estimator V2 – Used as the interactive environment to write, run, and visualize the experiments
  • Google Colab + Jupyter Notebook – Used as the interactive environment to write, run, and visualize the experiments
  • NumPy - For numerical operations such as rounding and calculating relative errors
  • Matplotlib - For visualizing outputs

Simple Protocol

Overview

  • This project implements a basic TCP-based client-server protocol for controlling a virtual light.
  • The protocol uses structured headers and logs all communication between a client and server to log files.
  • Communication is based on a simple protocol:
    • All messages start with a 12-byte header: version (4 bytes, integer), msg_type (4 bytes, integer), msg_length (4 bytes, integer)
    • This is followed by a UTF-8 encoded message body<
  • Message Types:
    • 0: HELLO message
    • 1: LIGHTON command
    • 2: LIGHTOFF command

Tech Stack

  • Python - Main implementation of client-server communication
  • socket module - TCP/IP-based socket programming
  • struct module - Packs/unpacks custom binary headers
  • logging module - Logs events, messages, and errors to console and file
  • argparse - command-line arguments for client/server setup
  • pcap - Analyze packets exchanged over the network

Network Packet Handling

Overview

  • This project demonstrates custom network packet construction and transmission between a client and server using Python sockets.
  • The packet includes a structured header and supports various payload types (integer, float, string).
  • Client.py:Sends custom packets to the server.
  • Server.py:Receives packets, parses the header and payload, and echoes the information back to the client.
    • All messages start with a 12-byte header: version (4 bytes, integer), msg_type (4 bytes, integer), msg_length (4 bytes, integer)
    • This is followed by a UTF-8 encoded message body<
  • Package FormatEach packet has:
    • Header (5 bytes total)
      • version (1 byte)
      • header_length (1 byte)
      • service_type (1 byte): 1 = integer payload, 2 = float payload, 3 = string payload
      • payload_length (2 bytes, unsigned short
    • Payload (variable):
      • Encoded based on service_type

Tech Stack

  • Python - Language for network logic and data handling
  • socket module - Enables TCP-based client-server communication
  • struct module - Handles packing/unpacking of binary headers and typed payloads
  • UTF-8 & Binary - Supports transmission of strings, integers, and floats over the wire
  • argparse - Provides a command-line interface for client configuration
